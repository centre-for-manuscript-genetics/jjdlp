Advanced search docs

Congratulations, you are actually willing to read the docs in order to constitute useful search queries.

The search engine behind this website is Elasticsearch, implemented through the Python library Haystack. The input fields provide the choice of the different sections of the website. Choosing "Sources" will provide iteration over all the sources, "Notes" over notes, etc. Leaving all field blank allows searching throughout every field. Within "Sources", title and author have received extra boost attributes, allowing them to rank higher on hit. The searchbox allows any query, and iterates over its indexed database via contain statements. This means that partial searches will not be valid, and queries such as "sulli" or "kel" will, for various reasons, not return The Book of Kells by Edward Sullivan. Furthermore, the engine interprets multiple words as AND statements (e.g. "sullivan kells" is interpreted as "sullivan AND kells"). OR statements are not (yet) valid ("sullivan OR gwynn" will return no results).

Click the type icon to search all of this type.
